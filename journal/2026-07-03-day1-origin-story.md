# day 1 — how this whole thing even started

okay so before any of the network stuff, VLANs, OPNsense, any of that — this actually
started way back with Arduino. I was messing around with Arduino for other projects and 
at some point just naturally started looking for something more powerful for whatever the
next experiment was going to be. that's how I landed on Raspberry Pi.

and Pi turned out to be a whole different thing compared to Arduino, way more capable, so
I spent a good bit of time just researching what people actually do with these things.
one of the first real ideas that stuck was setting up a network-wide DNS server — basically
something that filters ads/trackers for every device on the network instead of doing it
per-device. that idea alone kind of pulled me into the whole self-hosting rabbit hole.

but then I actually looked at Pi pricing and it wasn't cheap for what you get. and I
remember thinking — if I'm going to spend this much on a single board computer anyway,
why not look at getting a proper, slightly better spec'd used computer instead. more power,
probably similar cost, and it opens up doing a lot more than just one single-purpose thing.

that's the thought that got me into used PCs and server parts.

budget was a real constraint though so this wasn't a quick decision, spent a few months
just researching before actually buying anything. read up on what to look for in used
enterprise gear, what's worth buying used vs new, which parts fail vs which basically never
do. ended up feeling fine about buying most of it secondhand — figured used server parts
are usually fine since there's less moving parts to begin with compared to consumer stuff,
and enterprise gear is generally built to run 24/7 anyway so a few years of use isn't
really a dealbreaker.

only things I bought new were the server case and the PSU — everything else, all used,
picked up off OLX and Facebook Marketplace over time.

so that's really where it started. so I started hunting.
